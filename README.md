## ✨ ฟังก์ชันเด็ดๆ ที่เรามี

### 🛍️ การจัดการสินค้าและแคตตาล็อก

* **แสดงรายการชุดสวยๆ:** ชุดแต่ละชุดมาพร้อมรูปภาพคุณภาพดี คำอธิบายโดยละเอียด และราคาเช่าที่ชัดเจนตามระยะเวลา
* **จัดหมวดหมู่เข้าใจง่าย:** เราแบ่งชุดออกเป็นหมวดหมู่ต่างๆ (เช่น "ชุดราตรี", "ชุดลำลองสุดชิค", "คอลเลกชันฤดูร้อน") ให้คุณเลือกดูได้ง่ายๆ แต่ละหมวดหมู่ก็มีหน้าของตัวเองพร้อมรูปปกสวยๆ ด้วยนะ
* **โซนพิเศษห้ามพลาด:**
    * **สินค้ามาใหม่ (New Arrivals):** อัปเดตชุดใหม่ล่าสุดให้คุณไม่ตกเทรนด์
    * **สินค้ายอดนิยม (Popular Rentals):** รวมชุดฮิตที่ใครๆ ก็เช่า หรือได้รีวิวดีๆ
* **เจาะลึกรายละเอียดชุด:** ทุกชุดมีหน้าของตัวเอง บอกข้อมูลครบถ้วน ทั้งไซส์ที่มีให้เลือก แพ็กเกจเช่าต่างๆ และปุ่ม "หยิบใส่ตะกร้า"
* **ค้นหาชุดที่ใช่:** สามารถค้นหาชุดที่ต้องการภายในหมวดหมู่นั้นๆ ได้ *(อนาคต: อาจจะมีฟิลเตอร์ขั้นสูงให้เลือกตามไซส์ สี แบรนด์ หรือช่วงราคา)*

### 🚶‍♀️ ประสบการณ์ผู้ใช้งาน (User Journey)

1.  **ก้าวแรกสู่ MindVibe:**
    * **หน้าต้อนรับ (`/`):** แนะนำ MindVibe พร้อมปุ่มชวนให้ "เข้าสู่ระบบ" หรือ "สมัครสมาชิก"
    * **สมัครสมาชิก (`/signup/`):** สร้างบัญชีผู้ใช้ใหม่ ปลอดภัยหายห่วง
    * **เข้าสู่ระบบ (`/login/`):** สำหรับสมาชิกเก่าที่คิดถึงกัน
2.  **ท่องโลกแฟชั่น:**
    * **หน้าหลัก (`/home/`):** พบกับ Banner โปรโมชันเด็ดๆ หมวดหมู่แนะนำ และลิงก์ไปยังชุดใหม่/ชุดฮิต
    * **หน้าหมวดหมู่สินค้า (`/category/<ชื่อหมวดหมู่>/`):** รวมชุดทั้งหมดในหมวดหมู่นั้นๆ พร้อมช่องให้ค้นหาเพิ่มได้
3.  **เลือกชุดที่ชอบ & จัดการตะกร้า:**
    * **หน้ารายละเอียดสินค้า (`/clothing/<รหัสสินค้า>/`):** ให้คุณเลือกแพ็กเกจเช่าและไซส์ก่อนหยิบใส่ตะกร้า
    * **ตะกร้าสินค้า (`/cart/`):** ตะกร้าสินค้าแบบ Session-based ที่คุณสามารถเข้ามาดูชุดที่เลือกไว้ จะแก้ไข (ถ้าทำได้) หรือลบออกก็ได้
4.  **สั่งเช่า & ชำระเงิน:**
    * **หน้ายืนยันการเช่า (`/checkout/`):**
        * สรุปรายการชุดทั้งหมดที่คุณจะเช่า
        * กรอกข้อมูลการติดต่อและที่อยู่สำหรับจัดส่ง (หรือนัดรับ)
        * แจ้งรายละเอียดช่องทางการชำระเงิน (ตอนนี้เป็นแบบโอนผ่านบัญชีธนาคาร)
        * อัปโหลดหลักฐานการโอนเงิน (สลิป) หรือจะแนบเป็นลิงก์ก็ได้
    * **ยืนยันคำสั่งเช่า & ขอบคุณ (`/order/place/` และ `/order/thank-you/`):** แจ้งว่าการสั่งเช่าของคุณสำเร็จแล้ว พร้อมข้อความขอบคุณน่ารักๆ
* **ออกจากระบบ (`/logout/`):** จบเซสชันอย่างปลอดภัย

### ⚙️ ระบบหลังบ้านสำหรับแอดมิน

* **Django Admin Panel (`/admin/`):**
    * **จัดการสินค้า (Clothing):** เพิ่ม ลบ แก้ไขข้อมูลชุดต่างๆ ทั้งชื่อ คำอธิบาย รูปภาพ ราคาเช่า (3, 5, 7 วัน) สถานะพร้อมเช่า และหมวดหมู่ (slug จะถูกสร้างให้อัตโนมัติ)
    * **จัดการหมวดหมู่ (Category):** สร้างและจัดการหมวดหมู่สินค้า ทั้งชื่อ slug รูปปก คำอธิบาย และสถานะการแสดงผล (มีบอกด้วยว่าหมวดหมู่นี้มีสินค้ากี่ชิ้น)
    * **จัดการ Hero Banners:** ควบคุม Banner สวยๆ ที่หน้าแรก รองรับทั้งรูปภาพและวิดีโอ สามารถใส่ลิงก์ และจัดลำดับการแสดงผลได้

## 🛠️ ล้วงลึกเรื่องเทคนิค

### 💻 เทคโนโลยีที่เราเลือกใช้

* **ฝั่ง Backend:** Django 4.2+ (เขียนด้วยภาษา Python)
* **ฝั่ง Frontend:** HTML5, CSS3, JavaScript (ร่วมกับ Django Templating Language)
* **ฐานข้อมูล (Database):**
    * ตอนพัฒนา: SQLite3 (ตัวพื้นฐานของ Django)
    * ตอนใช้งานจริง (แนะนำ และมี config ใน Docker ให้): PostgreSQL 13
* **การทำ Containerization:** Docker และ Docker Compose
* **ไลบรารี Python สำคัญๆ:**
    * `Pillow`: สำหรับจัดการรูปภาพ
    * `psycopg2-binary`: ตัวเชื่อมต่อ PostgreSQL กับ Python
    * `django-humanize`: ช่วยให้ข้อมูลดูเป็นมิตรกับผู้ใช้มากขึ้น (เช่น ใส่จุลภาคในตัวเลข)

### 🗃️ แอบดูโครงสร้างฐานข้อมูล (Models หลัก)

ข้อมูลหลักๆ ของเราถูกกำหนดโครงสร้างไว้ในไฟล์ `clothes/models.py`:

* **`Category`**: เก็บข้อมูลเกี่ยวกับหมวดหมู่สินค้า (ชื่อ, slug, รูปภาพ, คำอธิบาย, สถานะการใช้งาน)
* **`Clothing`**: แทนข้อมูลชุดแต่ละชุด (เชื่อมโยงกับ `Category`, มีชื่อ, slug, คำอธิบาย, รูปภาพ, ราคาเช่าสำหรับ 3/5/7 วัน, สถานะพร้อมเช่า)
* **`HeroBanner`**: จัดการเนื้อหาสำหรับ Hero Banner ในหน้าแรก (หัวข้อ, คำโปรย, ประเภท (รูป/วิดีโอ), ไฟล์สื่อ, ลิงก์, สถานะการใช้งาน, ลำดับการแสดงผล)
* **`Cart` & `CartItem`**: (ปัจจุบันระบบตะกร้าใช้ Django sessions เป็นหลัก แต่มี Model เตรียมไว้ เผื่ออนาคตอยากเก็บข้อมูลตะกร้าลงฐานข้อมูลถาวร)
    * การทำงานของตะกร้าสินค้าในปัจจุบัน (ดูได้จาก `clothes/views.py` ตรง `request.session.get('cart', {})`) จะใช้ Session framework ของ Django ในการเก็บข้อมูลตะกร้าไว้ใน Session ของผู้ใช้แต่ละคนโดยตรง

### ❤️ หัวใจสำคัญของ Django App (clothes)

* **`config` (ระดับโปรเจกต์):**
    * `settings.py`: ไฟล์ตั้งค่าหลักของโปรเจกต์ ทั้ง `INSTALLED_APPS`, `MIDDLEWARE`, `DATABASES`, `STATIC_URL`, `MEDIA_URL`, รวมถึง URL สำหรับ redirect หลัง login/logout
    * `urls.py`: ไฟล์กำหนด URL หลักของโปรเจกต์ แล้วค่อยส่งต่อไปให้ `urls.py` ของแต่ละแอป
* **`clothes` (ระดับแอปพลิเคชัน):**
    * `models.py`: กำหนดโครงสร้างฐานข้อมูล (ตามที่อธิบายไปข้างบน)
    * `views.py`: เป็นส่วนของ Logic หรือการทำงานต่างๆ ที่ตอบสนองต่อ request จากผู้ใช้ และแสดงผลเป็นหน้าเว็บ เช่น:
        * หน้าต้อนรับ, หน้าหลัก, หน้ารายละเอียดสินค้า
        * ระบบสมาชิก (สมัคร, ล็อกอิน, ล็อกเอาต์)
        * หน้ารายการสินค้าตามหมวดหมู่
        * หน้าสินค้ามาใหม่ และสินค้ายอดนิยม
        * การจัดการตะกร้า (เพิ่ม, ดู, ลบ)
        * ขั้นตอนการสั่งเช่า (แสดงฟอร์ม, ยืนยันออเดอร์, หน้าขอบคุณ)
    * `urls.py`: กำหนด URL ย่อยๆ ที่เกี่ยวกับแอป `clothes` นี้โดยเฉพาะ ว่า URL ไหนจะไปเรียก view ฟังก์ชันไหน
    * `admin.py`: ปรับแต่งหน้า Admin ของ Django ให้จัดการ Model ต่างๆ ของแอป `clothes` ได้ง่ายขึ้น
    * `templates/`: โฟลเดอร์เก็บไฟล์ HTML ทั้งหมด โดยมีโฟลเดอร์ย่อย `clothes` ข้างในอีกที และมี `base.html` เป็น Template หลักที่หน้าอื่นๆ มา extend ต่อ
    * `static/clothes/images/`: เก็บรูปภาพที่ใช้ใน Template โดยตรง (เช่น รูป Banner, รูปประกอบสไตล์ต่างๆ)

## 🚀 มาเริ่มใช้งานกันเลย!

### 📋 สิ่งที่ต้องมีก่อนเริ่ม

* Python 3.8 ขึ้นไป
* pip (โปรแกรมติดตั้งแพ็กเกจ Python)
* Virtualenv (แนะนำให้ใช้ เพื่อแยกสภาพแวดล้อมของแต่ละโปรเจกต์)
* Docker และ Docker Compose (ถ้าอยากลองรันแบบ Container)
* Git (สำหรับ Clone โปรเจกต์)

### 🛠️ ขั้นตอนการติดตั้งสำหรับพัฒนาโปรเจกต์ (Local Development)

1.  **Clone โปรเจกต์มาไว้ที่เครื่องของคุณ:**
    ```bash
    git clone <URL ของ Repository คุณ>
    cd dsi202_2025
    ```

2.  **สร้างและ Activate Virtual Environment (แนะนำมากๆ):**
    ```bash
    python -m venv venv
    # สำหรับ Windows:
    venv\Scripts\activate
    # สำหรับ macOS/Linux:
    source venv/bin/activate
    ```

3.  **ติดตั้ง Dependencies ทั้งหมดที่โปรเจกต์ต้องการ:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **สั่ง Migrate Database (สร้างตารางตาม Models ที่เราออกแบบไว้):**
    ```bash
    python manage.py migrate
    ```

5.  **สร้าง Superuser (บัญชีผู้ดูแลระบบสูงสุด สำหรับเข้าหน้า Admin):**
    ```bash
    python manage.py createsuperuser
    ```
    (แล้วทำตามขั้นตอนที่ปรากฏขึ้นมา เช่น ตั้งชื่อ username, email, password)

6.  **รัน Development Server:**
    ```bash
    python manage.py runserver
    ```
    จากนั้นเปิดเว็บเบราว์เซอร์แล้วไปที่ `http://127.0.0.1:8000/` (หรือ URL ที่แสดงใน Terminal)

### 🐳 รันโปรเจกต์ผ่าน Docker (สำหรับคนชอบความเร็ว)

ถ้าอยากลองรันโปรเจกต์ผ่าน Docker แบบเท่ๆ (ต้องติดตั้ง Docker และ Docker Compose ก่อนนะ):

1.  **Build และ Start Containers:**
    ```bash
    docker-compose up --build
    ```
    เมื่อทุกอย่างพร้อม โปรเจกต์จะสามารถเข้าถึงได้ผ่าน `http://localhost:8000`
